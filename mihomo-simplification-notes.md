# Mihomo 简化记录

## 目标

目标是整理出一份可同时用于 OpenClash 和 macOS 的 Mihomo 配置。

当前先只关注两块核心内容：

- `proxy-groups`
- `rules`

第一阶段先做结构简化，不处理平台专属字段。

## 当前问题

导入进来的配置能力很强，但代理组的可见层级偏深，日常使用时不够直接。

目前复杂度主要来自这几类问题：

- 地区组和业务组都拆得比较细
- 中转、低延迟、流量型线路被做成了一层显式选择
- DNS、NTP、保护类分组混入了主操作层
- AI 和媒体类分组拆分粒度偏细，超过了日常切换需要

## 简化方向

### 1. 缩小用户日常需要操作的范围

目标不是减少功能，而是减少平时真正需要点开的组。

理想状态是：常用可见组控制在 5 到 8 个左右。

### 2. 保留一个小而稳定的基础层

基础层只保留最常用的线路入口：

- `Manual`
- `Auto`
- `HK`
- `JP`
- `SG`
- `US`
- 可选：`TW`、`UK`

其他低频国家可以继续存在，但优先放进 `Manual`，而不是都成为顶层入口。

### 3. 保留一个小而清晰的业务层

第一版业务层只保留高频、真正需要独立控制的类别：

- `Default`
- `AI`
- `YouTube`
- `Streaming`
- `Social`
- `Apple`
- `Microsoft`
- `Dev`
- `Finance`

能合并的尽量先合并。

例如：

- OpenAI、Claude、Gemini、Copilot、Grok 先合并为一个 `AI`
- 一些单独媒体组优先并入 `Streaming`，除非后面确认确实需要单独控制

### 4. 基础设施组尽量下沉

DNS、NTP、防劫持、保护类分组可以保留，但不应占据主层级的操作心智。

它们更适合作为规则引用的内部组，而不是主要用户入口。

### 5. 让规则出口更收敛

规则最终应尽量汇聚到较少的目标组，而不是把每个小业务都拆成单独出口。

建议的规则流向顺序：

1. 局域网 / 私有地址
2. 拒绝 / 防劫持 / 保护
3. 关键业务分类
4. 中国大陆直连
5. 其余流量回落到 `Default`

## 代理组分层草案

### 第一层：基础组

- `Manual`
- `Auto`
- `HK`

### 第二层：主业务组

## 当前阶段不做的事

这一轮先不处理：

- Clash for Windows 适配
- OpenClash 专属字段清理
- macOS 专属字段清理
- 最终命名优化
- 最终 DNS 行为设计

## 下一步

下一步应基于当前仓库里的实际配置文件，单独整理一份简化版 `proxy-groups` 草稿文件，不直接替换现有配置。

## Proxy Groups 第一阶段范围

这一阶段只做一件事：

- 简化国家线相关的 `proxy-groups`

明确不动的部分：

- 业务分组保留
- DNS 分组保留
- 规则保留
- 现有业务语义保留

这一步的目标不是重构整份配置，而是把现有业务组下面引用的“国家选择层”替换成一套更简单的结构。

## 当前确认的简化方案

国家选择层以当前 `config.yaml` 为参考，保留其命名习惯和使用方式，但压缩层级。

### 常用国家

保留四个高频国家组：

- `香港`
- `美国`
- `新加坡`
- `日本`

### 每个常用国家组内置一个自动选择

不新增一个独立的“常用国家自动”顶层组。

而是在每个常用国家组内部，放一个对应国家的 `url-test` 自动选择项，并将它作为默认首项。

也就是说，每个国家组本身仍然是手动选择组，但组内第一项是该国家自己的自动选择。

### 其他国家

其他国家不作为第一阶段重点，不新增复杂自动层。

如果当前 `config.yaml` 里已有相关国家入口，可以暂时保留；但业务组后续优先引用新的四国结构。

## 替换原则

不是删除现有业务组，而是把它们下面原来引用的各类国家线路入口，逐步替换为新的国家层。

也就是说：

- 保留现有 `AI`、`YouTube`、`Apple`、`Microsoft`、`开发` 等业务组
- 保留现有 `DNS` 和其他底层功能组
- 只把这些组内部的国家类候选项替换成新的国家选择结构

## 第一阶段预期结构

建议形成这样一层新的国家线结构：

- `香港`：`select`
- `美国`：`select`
- `新加坡`：`select`
- `日本`：`select`

每个国家组内部结构类似：

- 第一项：该国家的自动选择组，类型为 `url-test`
- 后续项：该国家筛选后的具体节点

例如：

- `香港`
  - `香港自动`
  - 若干香港节点

- `美国`
  - `美国自动`
  - 若干美国节点

如果 `config.yaml` 当前还有：

- `手动选择`
- `故障转移`

则这两个组是否保留，后续要结合当前文件逐项看，但不影响上面的国家主结构先成立。
